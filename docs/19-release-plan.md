# Release Plan

1. **Sprint 00** — setup (tooling, env, CI, scaffolding).
2. **01** — auth + business onboarding.
3. **02** — services + availability.
4. **03** — public booking flow.
5. **04** — jobs + calendar + customers.
6. **05** — team + crew view.
7. **06** — payments + notifications.
8. **07** — dashboard + settings.
9. **08** — beta hardening (errors, analytics, Sentry, perf, a11y, tests).

**Launchable** when the core flow works end-to-end with auth/permissions, public booking, schedulable/completable jobs, payment status, email confirmations, error handling, key tests, and analytics/error tracking installed.
